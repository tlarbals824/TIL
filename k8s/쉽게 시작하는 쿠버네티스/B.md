# 보강 B

## 오브젝트 vs 리소스

쿠버네티스의 오브젝트는 "클러스터 안에 실제로 존재하는 개별 엔티티"이고, 리소스는 "오브젝트들을 다루는 API 이름/엔드포인트"입니다.

### 오브젝트

* 클러스터 안에 실제로 만들어지는 실체
* 사용자가 yaml로 spec을 정의하면, 원하는 상태가 되고, 쿠버네티스가 이를 만족하는 실제 상태를 유지하려합니다.

### 리소스

* 쿠버네티스 API에서 사용하는 이름/엔드포인트
* 예로는 pods, deployments, services 같은 api 리소스 타입, `/api/v1/namespaces/default/pods` 같은 REST 경로
* 하나의 리소스 타입이 여러 오브젝트 인스턴스를 담는 컬렉션 역할을 함

## default 설정

쿠버네티스에서 default 설정은 보통 기본 네임스페이스(default namespace)나 리소스의 기본값을 의미합니디ㅏ.

### 기본 네임스페이스

* 클러스터 초기 생성 시 자동으로 만들어지는 네임스페이스 중 하나, 네임스페이스를 지정하지 않고 명령을 실행하면 default 네임스페이스가 기본 작업공간으로 사용됩니다.

### 리소스 요청/제한

* 리소스 선언에서 값을 지정하지 않을 경우 자동으로 기본값(default)을 채워 넣어줍니다.
